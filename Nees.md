# 系统首页统计展示功能设计 README

## 一、文档说明

本文档针对  操作卡培训考核管理系统，基于现有数据库结构，设计首页需展示的统计数据，实现首页数据可视化与高效操作入口。

## 二、核心设计原则

1. **核心数据优先**：聚焦培训考核核心业务，优先展示管理员 / 运营人员最关注的宏观指标；
2. **多维度拆分**：按「业务模块 + 系统运营」拆分数据，兼顾业务场景与系统管理；
3. **交互便捷性**：所有统计项支持钻取跳转，新增快捷操作 / 待办提醒，提升操作效率；
4. **可视化展示**：结合数字、趋势图、占比图、排名榜等形式，直观呈现数据。

## 三、首页功能模块设计

### 3.1 核心统计看板（顶部核心区域，大卡片可视化）

|     统计项      |                 数据来源表                 |                      展示形式                      |                   补充说明                    |

|   操作卡总数    |               hlvr_sop_card                | 总数字 + 环形图（VR/PC 类型、培训 / 考核模式占比） |        统计表中总数量，按类型维度拆分         |

## 四、交互与样式要求

1. **数据钻取**：所有统计数字支持点击跳转至对应明细页面；

2. **时间筛选**：核心统计区域增加「今日 / 本周 / 本月 / 自定义」时间筛选器；

3. 可视化样式

   ：

   - 核心指标：大数字 + 趋势图 / 环形图；
   - 占比类：饼图 / 环形图；
   - 趋势类：折线图 / 柱状图；
   - 排名类：横向柱状图；

4. **个性化配置**：管理员可自定义首页展示模块、调整模块顺序、隐藏无需关注的统计项。

## 五、开发落地说明

1. 所有统计项需关联文档中指定的数据库表 / 字段，保证数据准确性；
2. 优先实现核心统计看板 + 业务模块数据，再补充系统运营数据与快捷功能；
3. 可视化组件建议使用 ECharts/Element UI 等成熟框架，降低开发成本；
4. 数据加载需做异步处理，避免首页卡顿；
